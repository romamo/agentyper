# Session — Fix Missing Mutating Flags in Sub-app Commands

**Date:** 2026-04-07  
**Workspace:** agentyper  
**Model:** Claude Sonnet 4.6

---

## 1. Goals

- Fix a bug where `--dry-run` and `--idempotency-key` flags were not added to commands mounted via `add_agentyper` (sub-apps).

## 2. Work Done

- Identified root cause in `src/agentyper/_internal/_app.py`: `_mount_into` was missing the `if cmd_info.mutating:` block that adds `--dry-run` / `--idempotency-key`, present in `_build_parser_internal`.
- Also found that `_mount_into` did not recursively handle `self._sub_apps`, meaning deeply-nested sub-apps would silently vanish.
- Fixed both issues in `_mount_into` (lines ~424–449): added the mutating flags block and added the recursive `_sub_apps` loop mirroring `_build_parser_internal`.
- Added regression test `TestAdvancedFeatures::test_dry_run_flag_in_sub_app` in `tests/test_agentyper.py` covering `price fetch AAPL --dry-run`.
- Ran full test suite — 33 tests pass.

## 3. Decisions Made

| Decision | Rationale |
|----------|-----------|
| Mirror `_build_parser_internal` logic exactly in `_mount_into` | Keeps both paths consistent; reduces drift risk |
| Also fix missing `_sub_apps` recursion in `_mount_into` | Discovered while reading the method; silent data loss for nested sub-apps |

## 4. Decisions Refused / Alternatives Rejected

| Rejected Approach | Reason |
|-------------------|--------|
| Extract shared helper for mutating flags | Only two call sites; premature abstraction per project rules |
