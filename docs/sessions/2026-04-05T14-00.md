## 3. Decisions Made

| Decision | Rationale |
|----------|-----------|
| All 5 envelope keys always present | Spec (REQ-F-004): "The schema of this envelope MUST NOT vary based on result count, command type, or success/failure state" |
| `echo()` plain strings → stderr in JSON mode | Preserves machine-readable stdout contract; prose like "Aborted." must not interleave with JSON |
| `_invoke_fn` split from `_call_fn` | Idempotency cache needs the raw return value before rendering; callbacks don't participate |
| Thread-local for warnings/pagination/timing | Commands run in the same thread as dispatch; module-level would conflict across test invocations |
| Module-level `_idempotency_cache` | Session-scoped deduplication is correct; in-process cache matches the spec's session semantics |
| Binary search for size cap | Iterative removal is O(n²) for large lists; binary search is O(n log n) and produces exact fit |
| SIGALRM-based timeout (Unix only) | Threads can't interrupt I/O-blocked code; `SIGALRM` is the only reliable per-process wall-clock mechanism on Unix; guarded with `try/except (ValueError, OSError)` for non-main-thread safety |
| `ResourceId` as opt-in type annotation | Spec says "command authors declare the argument type"; auto-applying to all strings would reject legitimate inputs (e.g. email addresses containing `@`) |
| `danger_level` defaults inferred from `mutating` | Backward compatibility; existing code using `mutating=True` silently gets `danger_level="mutating"` |
| `default_timeout_ms=0` (disabled) in `Agentyper` | Opt-in timeout; a 30s default would break long-running commands without author declaration |

## 4. Decisions Refused / Alternatives Rejected

| Rejected Approach | Reason |
|-------------------|--------|
| Enforce `effect` field at registration time (hard error) | Would break all existing mutating commands that pre-date this requirement; soft path (rely on `result()` helper) chosen |
| Auto-apply `ResourceId` validation to all string args | Would reject valid strings containing `#`, `?`, `@` (emails, tags, URLs) — must be opt-in per spec |
| Apply ANSI stripping to final JSON string with regex | Correct for `ensure_ascii=False` mode, but fragile; deep-recursive approach on the data structure is cleaner and strips before serialization |
| Send error responses (NOT_FOUND, CONFLICT) to stdout as envelope | Current design sends structured errors to stderr; changing stdout contract would break consumers that `json.loads(stdout)` on error paths |
| Thread-based timeout | Cannot interrupt I/O-blocked Python code; only `SIGALRM` (Unix) or `os._exit` path works reliably |
| Persistent (cross-session) idempotency cache | Requires external storage; not part of the framework responsibility; session-scoped is correct for in-process dedup |
