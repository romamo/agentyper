## 3. Decisions Made

| Decision | Rationale |
|----------|-----------|
| Keep timeout capability-based for `Agentyper(...)` parsers | The original goal was to avoid cluttering help output when timeout support is not relevant. |
| Preserve `--timeout` by default for `agentyper.run()` | A review pass identified a behavioral regression for single-function CLIs, so `run()` was adjusted separately. |
| Accept `--yes`, `--no`, and `--answers` consistently even when hidden from `--help` | This avoids blocking or parse failures when interaction is reached indirectly through helpers while still keeping help compact. |
| Keep interaction flags hidden from `--help` on non-interactive commands unless explicitly marked | This preserves the cleaner help output the user wanted without removing the defensive non-TTY path. |
| Use explicit `interactive=True` when authors want visibility despite indirect interaction | Static detection is best-effort only, so authors need a reliable override. |

## 4. Decisions Refused / Alternatives Rejected

| Rejected Approach | Reason |
|-------------------|--------|
| Remove `--timeout` from `agentyper.run()` by default | A review found this was a user-visible regression for existing single-function CLI behavior. |
| Treat hidden interaction flags as unsupported by the parser | The user wanted callers to be able to pass `--yes` even when interactivity was not explicitly declared, and this also avoids indirect-interaction failures. |
| Require `--yes` to remain visible in `--help` for completely non-interactive CLIs | The user clarified that backward compatibility was not required there, and hiding irrelevant flags better matched the intended help UX. |
