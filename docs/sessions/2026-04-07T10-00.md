## 1. Goals

- Implement support for variadic positional arguments (`nargs="*"`, `nargs="+"`) when a command parameter is annotated with `list[T]` and declared with `Argument()`.
- Expected: `bean fetch f1.beancount f2.beancount` instead of `bean fetch '["f1.beancount", "f2.beancount"]'`.

## 2. Work Done

- Read `src/agentyper/_internal/_app.py` and `src/agentyper/_internal/_params.py` to understand current argument-handling logic.
- Added `_list_inner_type(annotation)` helper function in `_app.py` that extracts the inner type `T` from `list[T]` or `Optional[list[T]]` annotations, returning `None` for non-list types.
- Updated `_add_argument()` in `_app.py`:
  - When the annotation is `list[T]`, uses `nargs="+"` (if `default=...`, i.e. required) or `nargs="*"` (if optional), with `type=_make_type_fn(inner_type)`.
  - Falls back to original single-value / JSON-array path for non-list annotations.
- Added `from pathlib import Path` at module level in `tests/test_agentyper.py` (required for `get_type_hints()` to resolve `Path` in test functions).
- Added 3 new tests in `TestAdvancedFeatures`:
  - `test_variadic_argument_required` ‚Äî `list[str]` with `Argument(...)`, expects `nargs="+"`
  - `test_variadic_argument_optional` ‚Äî `list[str]` with `Argument(None)`, expects `nargs="*"`
  - `test_variadic_argument_typed` ‚Äî `list[Path]` with `Argument(...)`, verifies inner type coercion
- All 37 tests pass.

## 3. Decisions Made

| Decision | Rationale |
|----------|-----------|
| Detect list annotation only in `_add_argument` (ArgumentInfo path), not in bare positional path | Keeps the existing JSON-array behaviour for plain positional `list[T]` params (no `Argument()`); only variadic shell syntax when `Argument()` is used explicitly. |
| Use `nargs="+"` for `default=...` and `nargs="*"` for optional | Mirrors argparse semantics: `"+"` requires at least one value; `"*"` allows zero. |
| Added `_list_inner_type` as a standalone helper rather than inlining | Avoids duplicating the union-unwrapping logic; keeps `_add_argument` readable. |
| Moved `from pathlib import Path` to module level in test file | `get_type_hints()` resolves annotations against module globals; local imports inside test methods are invisible to it. |

## 4. Decisions Refused / Alternatives Rejected

| Rejected Approach | Reason |
|-------------------|--------|
| Changing bare positional `list[T]` (no `Argument()`) to also use `nargs` | Would break the existing `test_list_parameter` test which passes JSON arrays as a single string; backward-incompatible change not requested. |
